    function applyFilters(){
      let filtered = [...products];

      // size (at least one match)

      if(sizes.length) filtered = filtered.filter(p=>p.size.some(s=>sizes.includes(s)));

  1. Skapa filen index.html och klistra in koden.

  2. Öppna filen i webbläsaren — du får en fullt fungerande produktkatalog med sök, filter, sort och snabbvy.

3) Nästa steg — två vanliga vägar beroende på behov

A) Vill du ha en färdig butik (betalningar, lager, orders)?

  • Rekommendation: WordPress + WooCommerce, eller Shopify.

  • Fördelar: betalningsadapters, leverans, orderhantering, plugin-ekosystem.

  • Hur: installera WordPress, välj ett e-handels-tema (t.ex. Astra, Storefront) och synkronisera produktbilder och variationer. Du kan använda ovanstående HTML/CSS som inspiration eller göra en custom frontend via ett child theme.

B) Vill du ha full kontroll / snabb frontend (headless)?

  • Använd Next.js eller Vue + Headless CMS (Contentful/Strapi) eller ett e-commerce-backend (Commerce.js, Medusa, Strapi + Stripe).

  • Fördelar: snabb, modern UX, kan använda samma komponenter som ovan (men som React-komponenter).

4) Tips för att få sidan att kännas som professionell butik (modena.se-stil)

  • Använd högupplösta produktbilder (min 2000px bred för produktvisning).

  • Sätt upp produktvarianter (storlek/colour) i backend och visa tydligt lagerstatus.

  • Implementera filtrering server-side för stora kataloger (inte bara klient-sida).

  • SEO: använd schema.org/Product, meta tags, bra produkttexter och url-struktur /product/herr/jackor/produktnamn.

  • Prestanda: lazy-load bilder (demo använder loading="lazy"), komprimera bilder WebP.

  • Betalningar: Stripe eller Klarna för Sverige. SSL krävs (HTTPS).

  • Frakt/integration: PostNord, DB Schenker eller annan lokal fraktleverantör via plugin eller API.

✅ Steg 4 — Aktivera GitHub Pages (gratis hemsida)

När index.html är uppladdad:

  1. Gå till Settings (fliken längst upp)

  2. I vänstermenyn scrolla ner till Pages

  3. Under Build and deployment välj:

  • Source: Deploy from a branch

  • Branch: main

  • Folder: / (root)

  1. Klicka Save

Efter 10–30 sekunder får du en gratis URL:

➡️ https://dittnamn.github.io/min-webbsop/

Det blir din live hemsida!

✅ Så här ser knappen ut när du ska ladda upp filen

När du är inne i ditt repo (som på din bild) så hittar du uppe till höger:

Add file ▼

När du klickar på den får du två val:

  • Create new file

  • Upload files ← det är denna du ska klicka på

Det står alltså Upload files, inte “uplåde fil”.

Här är hur knappen brukar se ut:

sql
Add file ▼
   ├─ Create new file
   └─ Upload files

🔽 Efter att du klickat “Upload files”

Då öppnas en sida där du kan:

  • Dra en fil in i rutan
    eller

  • Klicka på Choose your files

Där väljer du index.html.
